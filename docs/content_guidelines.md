# SchemeAtlas Content Generation Guidelines (SOP)

This document outlines the official Standard Operating Procedure for generating and validating content for Indian government schemes on the SchemeAtlas platform.

## 🔴 CRITICAL LINK RULES (NON-NEGOTIABLE)

1. **NEVER use placeholder links**:
   - No "#", "Not specific", or empty hrefs.
   - No fake or guessed URLs.

2. **Verified Links ONLY**:
   - If the official application link is NOT 100% verified, do NOT create a clickable link.
   - Instead, use the text: *"Official application link not available. Please visit the official ministry website."*

3. **Trusted Domains ONLY**:
   - Use only `.gov.in`, `nic.in`, or official state/central portals.
   - For all external links, include: `target="_blank"` and `rel="nofollow noopener"`.

4. **Zero Hallucination Policy**: Never hallucinate or guess URLs.

---

## 🟡 APPLY BUTTON LOGIC

- **IF official link exists** → Show: **"Apply on Official Website"**
- **IF official link does NOT exist** → Show: **"Check official website for application process"**

---

## 🟢 CONTENT QUALITY RULES

- **Readability**: Friendly, conversational, and simple. Use very short sentences.
- **Language**: Avoid formal, complex, or heavy words. Write like you are talking to a friend.
- **Efficiency**: Avoid repetition, fluff, and vague phrases like "Not specified" or "May vary."
- **Accuracy**: Provide the best-known accurate information or clearly explain specific conditions.
- **No Markdown Symbols**: DO NOT use `#` for headers or `**` for bold in the final article text. Use plain text or simple labels instead.
- **Tone**: Helpful, easy to understand, and practical.

---

## 🧠 E-E-A-T (Expertise, Experience, Authoritativeness, and Trustworthiness)

- Mention the **Official Authority** (Ministry/Department) responsible for the scheme.
- Include a verification footer at the end of every article:
  Source: <Official Ministry/Department Name>
  Last Verified: <Current Month, Year>

---

## 📊 STRUCTURE TO FOLLOW

Each scheme page must follow this exact 14-point structure. Use clear, plain labels for each section (No # symbols):

1. Title: Easy to understand, friendly.
2. Summary: 5–10 lines. Very simple overview.
3. What is the Scheme?: Purpose and Ministry details.
4. Key Benefits: Exact money (₹) or facilities. Use simple language.
5. Eligibility Criteria: Simple bullet points.
6. Who Should Apply: Real-life examples.
7. Who Should NOT Apply: Clear disqualification cases.
8. Documents Required: Simple list.
9. Selection / Approval Process: Step-by-step in plain words.
10. How to Apply: Simple steps anyone can follow.
11. Important Dates: If any.
12. Official Website / Application: 
    - Verified URL if it exists (No markdown link syntax, just the URL).
    - Plain text if it does not.
13. FAQs: 5–7 friendly questions and answers.
14. Pro Tips / Insights: Simple advice to help the user.

---

## 🔍 SEO OPTIMIZATION

- Natural keyword integration:
  - `<scheme name> eligibility`
  - `<scheme name> benefits`
  - `how to apply <scheme name>`
- Use proper heading hierarchy (H2, H3).
- Use bullet points to improve readability and "scannability."

---

## 🔁 SCALING RULES

- Each page must be UNIQUE in wording.
- Vary sentence structure and phrasing to avoid repetitive templates across 400+ pages.
